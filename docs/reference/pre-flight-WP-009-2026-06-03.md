
**Forbidden (any change fails the WP):**

- Game data binaries committed to git (any `*.exe`, `*.eng`, `*.dat`, `*.iso`, `*.ART`, `*.MMF`, `*.MMA`, `*.MMP`, `*.MMS`, `*.ini`) — copyright; safety-net in `.gitignore`. WP-009 does not produce any such files, but the rule applies as a backstop.
- **Captures from a locally-mounted Showdown ISO playthrough.** Capture source MUST be public YouTube footage. Captures taken from a running game on a mounted disc are screen-recordings of copyrighted game data the operator has played — outside the YouTube-public-footage fair-use posture WP-009 §Notes establishes. If the operator wants to capture from a playthrough, that's a different WP with a different copyright analysis.
- `engines/scooby/**` — Phase 2 territory; nothing here.
- `docs/formats/**` — WP-009 locks no format facts; no edits to `tgifile-art.md`, `scooby-exe.md`, `mmfw-container.md`, or `audio-archives.md`.
- `docs/execution-checklists/**` — no EC exists for WP-009 (inline checklist instead); do not author one mid-execution.
- `tools/**` — no tool scripts, no extracts, no cache changes. The cached `object.ini` and `asset-catalog.json` stay untouched.
- Capturing Phantom / Jinx / Case File frames in the same session — those are separate WPs each, regardless of how tempting cross-title comparison feels.
- Cutscene frames — Bink decoder handles cutscenes (Phase 3 / 4 work).
- UI-only screenshots (inventory screen, options screen) — explicitly out of scope per WP-009 §Scope → Out of scope.
- Animation-frame sprite captures beyond the single `OBJ_DAPHNE_A` (or substitute) anchor — Phase 4 inventory animation work.

**Exit check (must run before staging each of Commit A and Commit B):**

Any file in the output that doesn't match the allowed list = scope violation. Remediate with `git restore --staged`, not by widening the allowlist post-hoc.
